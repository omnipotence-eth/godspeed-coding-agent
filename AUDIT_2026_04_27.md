# Godspeed Coding Agent — Comprehensive Audit Report
**Date:** 2026-04-27
**Auditor:** OpenCode (kimi-k2.6)
**Scope:** Full codebase audit — architecture, security, dependencies, CI/CD, tests, and 2026 readiness

---

## Executive Summary

Godspeed is a **production-grade, security-first coding agent** that punches above its weight. The architecture is sound, the security model is best-in-class for open source, and the CI/CD pipeline meets professional standards. **~854 tests, 80% coverage gate, 5-stage CI, ruff-clean, type-checked.**

**Grade: A-** — World-class security and architecture. Held back by stale model references, one flaky test, and a few 2026-era stack gaps.

## 1. Architecture & Code Quality

### What's Excellent

| Area | Assessment |
|------|-----------|
| **Agent loop** | Hand-rolled async loop with speculative dispatch, parallel/serial tool execution, stuck-loop detection, auto-verification, auto-stash, pause/resume, cancel events. Matches patterns proven by leading open-source coding agents. |
| **Security model** | 4-tier permission engine (deny > dangerous > session > allow > ask > default), 71 dangerous command regex patterns, 4-layer secret protection (file deny-list → context cleaning → output filtering → audit redaction), hash-chained SHA-256 audit trail with fsync durability. |
| **Tool system** | 25 built-in tools with JSON Schema descriptors, risk levels, and Pydantic `ToolResult`/`ToolCall` models. Clean `Tool` ABC. |
| **LLM abstraction** | LiteLLM wrapper with fallback chains, model routing by task type (plan/edit/read/shell), rate-limit retry with exponential backoff + jitter, token counting, cost tracking, prompt caching. |
| **TUI** | prompt-toolkit + Rich, slash commands, key bindings, diff review prompts, permission dialogs, status HUD, themes. |
| **Config** | Pydantic v2 + pydantic-settings with YAML merge (global > project), env var override, field validators, model presets, routing shortcuts. |
| **Memory** | SQLite-backed persistent preferences, session events, correction tracking. |
| **Self-evolution** | GEPA-style trace analysis → LLM-guided mutations → LLM-as-judge fitness → safety gate. Hardware-aware Ollama model selection. |
| **Training pipeline** | Conversation logger → TrainingExporter (openai/chatml/sharegpt) → per-step reward annotations → benchmark suite. |
| **MCP support** | stdio + SSE transport, auto-adaptation to Tool ABC with HIGH risk level. |

| Severity | Issue | Location | Fix |
|----------|-------|----------|-----|
| **Medium** | `PermissionDecision.__eq__` accepts `str` but not other types safely — returns `NotImplemented` correctly, but the type stub is loose. | `security/permissions.py:30` | Add `@overload` or tighten type hints if using `typing_extensions`. |
| **Low** | `agent_loop` has 15+ optional callback parameters. This is a signature smell — consider a `LoopCallbacks` dataclass. | `agent/loop.py:58` | Bundle callbacks into a single `LoopCallbacks` dataclass to reduce parameter count and improve testability. |

## 2. Security Posture

### What's Excellent

- **Deny-first permission engine** with additive deny rules (project config cannot weaken global denies).
- **Dangerous command detection** covers recursive delete, disk format, pipe-to-shell, SQL DROP, git force push, sudo, netcat, eval/exec, fork bombs, container escape, and more.
- **Secret scanning** with 27 regex patterns + Shannon entropy analysis for high-entropy strings.
- **Audit trail** uses `os.fsync()`, fails closed on write errors, hash-chains every record, supports gzip compression.
- **Redaction** runs before audit writes and before LLM context assembly.
- **Session grants** have TTL expiry (1 hour) with thread-safe lock.
- **Plan mode** blocks all non-READ_ONLY tools.
- **Architect mode** separates planning model from execution model.

### Issues Found

| Severity | Issue | Location | Fix |
|----------|-------|----------|-----|
| **Medium** | `dangerous.py` does not detect `powershell -Command` or `pwsh -c` on Windows — a significant gap since Windows is a first-class platform. | `security/dangerous.py` | Add patterns for `powershell.*-Command`, `pwsh.*-c`, `Invoke-Expression`, `Start-Process`, `Remove-Item -Recurse -Force`. |
| **Medium** | `sudo` is blanket-denied, but `doas`, `pkexec`, and `runas` (Windows) are not covered. | `security/dangerous.py` | Add `doas`, `pkexec`, `runas` to privilege escalation patterns. |
| **Low** | Secret patterns do not include `nvapi-` (NVIDIA API key prefix), `sk-ant-` is present but `sk-ant-api03-` (newer format) may be longer. | `security/secrets.py` | Add `nvapi-[a-zA-Z0-9\-]{20,}` and review Anthropic key length. |
| **Low** | `dd if=` pattern is too broad — `dd if=file.img of=/dev/sdb` is dangerous, but `dd if=input.txt of=output.txt` is a legitimate copy. Pattern should anchor on output to `/dev/`. | `security/dangerous.py:23` | Change to `r"dd\s+if=.*of=/dev/"`. |
| **Low** | No detection for `base64 -d | bash` or `echo <b64> | base64 -d | bash` — common obfuscation vector. | `security/dangerous.py` | Add `base64.*\|\s*(ba)?sh` and `echo.*base64.*\|\s*(ba)?sh`. |
| **Info** | Secret entropy threshold (`3.5`) is not configurable. | `security/secrets.py` | Expose as `GodspeedSettings.security.entropy_threshold`. |

## 7. Specific 2026 Gaps

| Gap | Impact | Recommended Action |
|-----|--------|-------------------|
| **No MCP ACP support** | ACP (Agent Communication Protocol) is emerging as a successor to MCP in some ecosystems. | Monitor ACP adoption; add support if it gains traction beyond experimental. |
| **No structured output / `instructor` integration** | Godspeed uses JSON Schema tool definitions but doesn't validate LLM outputs with Pydantic models via `instructor` or `outlines`. | Add optional `instructor` integration for structured generation in planning / architect mode. |
| **No OpenAI Responses API support** | OpenAI's new Responses API (GPT-5) replaces Chat Completions for some features. LiteLLM may handle this, but verify. | Test `gpt-5.1` via LiteLLM and document any needed config changes. |
| **No native fp8 / Blackwell inference** | For local models on RTX 50-series, fp8 inference via TensorRT-LLM or vLLM fp8 would be a differentiator. | Add documentation for `ollama` + fp8 quants, or a `vllm` local provider guide. |
| **No agent-to-agent communication** | Sovereign-edge and other multi-agent systems need A2A protocol. | Evaluate Google A2A protocol for inter-agent communication. |
